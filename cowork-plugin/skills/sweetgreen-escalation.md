# Sweetgreen Escalation Paths

When helping employees with ServiceNow tickets, understand the escalation structure so you can set expectations.

## How Assignment Works

ServiceNow automatically routes tickets based on the category and subcategory. You don't need to manually assign tickets — just set the right category and ServiceNow's assignment rules handle the rest.

## Escalation Tiers

### Tier 1 — Service Desk
- **Handles**: Password resets, account unlocks, basic software questions, VPN issues
- **SLA**: First response within 4 hours, resolution within 1 business day
- **Auto-assigned for**: Most new incidents

### Tier 2 — Specialized Teams
- **Network Operations**: WiFi, VPN, firewall, network outages
- **Desktop Support**: Hardware issues, OS problems, software installations
- **Application Support**: SaaS app issues, integrations, data problems
- **SLA**: Resolution within 2-3 business days

### Tier 3 — Engineering
- **Infrastructure**: Server issues, cloud services, database problems
- **Security**: Security incidents, suspicious activity, data breaches
- **SLA**: Varies by severity

## What to Tell Employees

After creating a ticket, set these expectations:
- "You'll get email notifications as the ticket progresses"
- "The IT team typically responds within 4 hours during business hours"
- "If this is urgent and you need immediate help, call the IT help desk at [extension]"
- "You can check your ticket status anytime by asking me"

## When to Recommend Calling Instead

For these situations, recommend calling the help desk directly:
- Complete inability to work (laptop dead, can't log in at all)
- Security concerns (suspected breach, phishing attack)
- POS system down at a store during operating hours
- Issues affecting an entire location
